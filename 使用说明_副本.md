# 栖 · 日记 — V1 使用说明

一个干净、离线、跨平台的日记本。Mac / Windows 都能用——双击 `diary.html` 在浏览器里打开即可。

## 怎么打开

**最省事**：双击 `diary.html`，会用默认浏览器打开。建议用 **Chrome** 或 **Edge**（语音输入只在这两个上工作得最稳）。Safari 也能用，但不能语音。

**首次打开会问两个权限**：
- 📍 **位置**（用于自动记录地点和天气）—— 选「允许」
- 🔔 **通知**（用于提醒到点弹窗）—— 选「允许」

不允许也能用，只是这两个功能要手动操作。

---

## 7 个功能，对应你的 7 条需求

### 1. 自动时间、地点、天气 ✅
新建日记时会自动填入当前时间、定位、天气。地点和时间都可以手动改；右边的 ↻ 按钮可以随时重新获取。
- 天气数据来源：Open-Meteo（免费、无需 key）
- 地点解析：BigDataCloud 反向地理编码（中文）
- **离线状态下**：会用最近一次缓存的位置和天气；联网后再 ↻ 即可

### 2. 关键词 / 标签 + 搜索 ✅（重点）
- 在「关键词 / 标签」区域输入，**回车** 或 **逗号** 添加
- 输入时自动从历史标签里联想（按使用频率排序）
- 标签**非强制**，但日记本身会根据内容里出现的词，自动建议加哪些已有标签
- 顶部搜索框：**全文搜索**——标题、正文、标签、表格内容、提醒内容、地点全部能搜到
- 左边栏顶部的标签云：点一下就按那个标签筛选；可以多选叠加
- 快捷键：**⌘K / Ctrl+K** 跳到搜索框

### 3. V1 离线，V2 同步 ✅
- 数据全部存在浏览器的 IndexedDB 里，**完全离线**
- 浏览器数据不会消失（除非你手动清空浏览器数据）
- 每条日记右下角有「导出」按钮，可以导出成 Markdown
- **V2 同步路线图**（在文末）

### 4. 文字 + 表格 + 公式 ✅
点「+ 添加表格」插入一张表。每张表：
- 默认 3 列（项目 / 金额 / 备注），可以加列、加行
- 表头和单元格都能直接改
- 「∑ 求和行」一键加上自动求和的合计行
- 一张日记可以放多张表（比如：购买记录、银行账户、卡路里…）

**支持的公式**（在单元格里以 `=` 开头）：
| 公式 | 例子 | 用途 |
|---|---|---|
| `=A1+B2` | `=B1+B2+B3` | 加减乘除 |
| `=B1*(1+C1)` | 含税金额 | 算术混算 |
| `=SUM(A1:A10)` | 区间求和 | 总额 |
| `=AVG(B1:B10)` | 平均值 | 月均消费 |
| `=MIN/MAX/COUNT(...)` | 最值/计数 | 统计 |

字体：用系统中文字（PingFang / 微软雅黑），干净、商务感。

### 5. 语音输入 ✅
正文右上角有个🎤按钮——点一下开始录，再点一下停止。
- 中文识别（普通话）
- 识别结果会**追加**到正文，不会覆盖
- 只在 **Chrome / Edge** 上工作；不支持的浏览器按钮会自动隐藏

### 6. 上传照片 ✅
点「+ 添加照片」或直接把图片**拖进编辑区**。
- 每条日记最多 **5 张**
- 自动压缩到 1280px、JPEG 80%（保证文件大小可控）
- 点照片可放大查看；hover 显示删除按钮

### 7. 提醒事项 ✅
点「+ 添加提醒」加一条提醒：选时间 + 写内容。
- 顶部🔔图标显示有几条到期未完成的提醒
- 点🔔展开，看所有未完成的提醒，按时间排序
- 每分钟自动检查一次；到点会弹**系统通知**（前提是你授权了通知）
- 完成后勾掉 checkbox

> ⚠️ 限制：提醒只在你**打开过这个网页**时才能弹通知。如果你关掉所有浏览器，到点不会响。这是离线 Web 应用的天然限制——V2 用客户端打包后可以解决。

## 快捷键

| 快捷键 | 功能 |
|---|---|
| ⌘N / Ctrl+N | 新建日记 |
| ⌘K / Ctrl+K | 跳到搜索框 |
| 标签输入框里 ↑/↓ | 在联想列表里选 |
| 标签输入框里 Backspace（空时）| 删掉最后一个标签 |

---

## 数据备份

数据全部在浏览器里。如果想搬到另一台电脑：
1. 在每条日记上点「导出」，会下载一份 `.md` 文件
2. V2 会做正式的「整库导出/导入」按钮

⚠️ **不要清浏览器的"网站数据"**，否则日记会丢。建议每隔一段时间导出一次重要的几条。

## V2 路线图（后续可以做的事）

按「实现难度 ÷ 你的需求强度」排序：

1. **整库导出 / 导入**（JSON 文件）—— 1 小时就能加
2. **打包成 Mac/Windows 桌面应用**（用 Tauri 或 Electron）—— 解决"关浏览器就不响"的提醒问题；同时可以读写本地文件
3. **同步到云**——三种方案，难度递增：
   - **iCloud Drive / OneDrive 文件夹同步**：把数据库文件放到云盘文件夹，最简单
   - **WebDAV / S3**：自己存到 NAS 或对象存储
   - **自建后端 + 账户系统**：最完整，也最贵
4. **手机端**：同一个 HTML 单文件其实在手机浏览器里也能用，但触屏体验一般。正式做的话用 PWA 或 React Native
5. **更多公式 / 图表**：折线图看消费趋势、饼图看分类
6. **语音转文字（离线版）**：用 Whisper Web 模型，不依赖浏览器 API
7. **加密**：本地数据库整体用密码加密（如果你介意别人在你电脑上点开浏览器就能看）
